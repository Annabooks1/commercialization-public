# Runtime Power Focused Stress with Driver Verifier's Concurrency Stress

<conditional_block> <conditions> <docset value="standalone"></docset> </conditions>

>[!NOTE]
You can find the latest version of this test documentation on MSDN at the following link:

-   <xref hlink="http://msdn.microsoft.com/en-us/library/windows/hardware/ee8d5a0b-628e-477b-9039-eb6824f42c3c">http://msdn.microsoft.com/en-us/library/windows/hardware/ee8d5a0b-628e-477b-9039-eb6824f42c3c</b>


</conditional_block>

This test performs I/O on different combinations of devices to alternate the devices between Active and Idle condition. Creating different combinations of devices being Active and Idle will help to ensure that the PEP logic is fully exercised.

## Parameters

| Parameter name                                   | Parameter description                              |
|--------------------------------------------------|----------------------------------------------------|
| <mark type="bullet_intro">DQ</b>                 | SDEL Device Query                                  |
| <mark type="bullet_intro">TestCycles</b>         | Number of test cycles                              |
| <mark type="bullet_intro">DelayBetweenCycles</b> | Delay time in milliseconds between each test cycle |
| <mark type="bullet_intro">IODuration</b>         | I/O duration in milliseconds                       |
| <mark type="bullet_intro">VerifierFlags</b>      | Driver Verifier flags to enable                    |
| <mark type="bullet_intro">TestTimeoutValue</b>   | Timeout value for the execution of te.exe          |
| <mark type="bullet_intro">OptTeCmdlineParams</b> | Optional commandline parameters for te.exe         |
