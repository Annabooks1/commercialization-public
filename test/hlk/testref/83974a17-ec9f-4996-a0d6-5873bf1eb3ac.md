This test measures the ability of a test computer to support debug operations using an xHCI controller, EHCI controller, serial port, 1394 controller, or a network adapter. The test validates that at least one debug port is available.

## Running the test

To complete this debug test, you must configure two computers. The computer that you are trying to certify is the <mark type="term">System under Test (SUT)</b>. The second computer is the <mark type="term">Host</b> on which the Windows debugger will run. These computers must meet the Windows HLK prerequisites and must be included in the same computer pool. For more information, see <xref hlink="http://msdn.microsoft.com/en-us/library/windows/hardware/jj124068.aspx">Windows HLK Prerequisites</b>. If the device being tested is an XHCI controller, an XHCI controller must be included in both computers. If the XHCI controller in the SUT has any user-accessible ports, the controller must have debug capability.

<note type="important">
You must have a cable that is appropriate for the debug transport that is being verified.

## Command syntax

 In the Parameters window **Transport** field, enter one transport type (NET, 1394, USB, or Serial). Based on the transport you entered, you must provide the following additional parameters.

These numbers are displayed in the USB host controller’s Properties window in Device Manager.</p>
<note type="important">

## Parameters

| Parameter name                                 | Parameter description                                                                                                                                                                                                                                  |
|------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <mark type="bullet_intro">Transport</b>        | Valid values are: net, usb, 1394, or serial                                                                                                                                                                                                            |
| <mark type="bullet_intro">NetHostIP</b>        | The IP address of the host machine                                                                                                                                                                                                                     |
| <mark type="bullet_intro">NetPort</b>          | Port number for kernel debugging. Port 50000 is recommended.                                                                                                                                                                                           |
| <mark type="bullet_intro">NetKey</b>           | The encryption key for kernel debugging. The default key is recommended.                                                                                                                                                                               |
| <mark type="bullet_intro">BusParams</b>        | Ethernet,1394, USB: Used only when more than one controller is present in the system under test. Enter bus params as b.d.f where b, d, and f are the bus number, device number, and function displayed in the relevant device manager properties page. |
| <mark type="bullet_intro">SerialTargetPort</b> | COM port number on the system under test                                                                                                                                                                                                               |
| <mark type="bullet_intro">SerialHostPort</b>   | COM port number on the host machine                                                                                                                                                                                                                    |
| <mark type="bullet_intro">SerialBaudRate</b>   | BAUD rate for serial communications                                                                                                                                                                                                                    |
| <mark type="bullet_intro">UsbTargetName</b>    | Any string is acceptable. The default is recommended.                                                                                                                                                                                                  |
| <mark type="bullet_intro">1394Channel</b>      | The common 1394 channel that will be used on both the host and system under test.                                                                                                                                                                      |
