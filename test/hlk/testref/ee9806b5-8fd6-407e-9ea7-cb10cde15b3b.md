# iSCSI HBA Ping Test (LOGO)

<conditional_block> <conditions> <docset value="standalone"></docset> </conditions>

>[!NOTE]
You can find the latest version of this test documentation on MSDN at the following link:

-   <xref hlink="http://msdn.microsoft.com/en-us/library/windows/hardware/4981a4bd-8aca-4ab7-ad67-8bdee56762b6">http://msdn.microsoft.com/en-us/library/windows/hardware/4981a4bd-8aca-4ab7-ad67-8bdee56762b6</b>


</conditional_block>

This test verifies that iSCSI initiators (HBA) can send and receive ping requests.

>[!NOTE]
There is no standard mechanism for initiating the sending of ICMP echo messages on an HBA. Therefore, manual interaction is required to complete this phase of the test.

## Running the test

Before you run the test, complete the test setup as described in the test requirements: <xref rid="p_hlk_test.iscsi_hba_testing_prerequisites">iSCSI HBA Testing Prerequisites</b>.

If this is the first time you are running an iSCSI HBA test, iSCSI Target test, or MPIO test over iSCSI bus type on the HLK client, or if there is no iscsihctconfig.ini under \[HLK Path\]\\JobsWorkingDir\\ on the HLK client, you will receive a pop-up dialog box to input iSCSI configuration information and/or HBA information. For more information about this dialog box, see “Running the test” in <xref rid="p_hlk_test.ca7ad4d0-6950-4e2d-bdfe-b80c7873ba90">iSCSI HBA Boot Test (LOGO)</b>.

This test returns Pass or Fail. To review test details, review the test log from the Windows HLK Manager.

## More information

This test operates in two phases:

1.  The first phase verifies that the HBA responds to ping requests by sending a ICMP echo packets to the HBA by using a network card in the test system.

2.  The second phase verifies the HBA can send ICMP echo packets to another address, which in this case is the network card in the test system.

To run this test:

1.  Run the test as a Driver Test Manager (DTM) job.

2.  Enter the required data in the configuration dialog box that appears when this job is scheduled.
