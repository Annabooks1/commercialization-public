# Oplocks Test

<conditional_block> <conditions> <docset value="standalone"></docset> </conditions>

>[!NOTE]
You can find the latest version of this test documentation on MSDN at the following link:

-   <xref hlink="http://msdn.microsoft.com/en-us/library/windows/hardware/df78b15c-2f8f-4cdf-9ebe-7b8781ceac84">http://msdn.microsoft.com/en-us/library/windows/hardware/df78b15c-2f8f-4cdf-9ebe-7b8781ceac84</b>


</conditional_block>

This automated test verifies the basic behavior of legacy opportunistic locks (oplocks) and enhanced oplocks. The legacy oplocks are Level2 (L2), Level1 (L1), Filter, and Batch. The enhanced oplocks are Read (R), Read-Write (RW), Read-Handle (RH), and Read-Write-Handle (RWH).

>[!NOTE]
For more information about the behavior of file systems (including oplocks), see <xref hlink="http://go.microsoft.com/fwlink/?LinkId=236047">File System Behavior in the Microsoft Windows Environment</b>.


The test runs variations for each of the six partitions that are described in the <mark type="bullet_intro">Running the test</b> section.

This test has the following variations:

-   <mark type="bullet_intro">Legacy\_Acquire\_Release.</b> Acquire a legacy oplock and then release it.

-   <mark type="bullet_intro">Enhanced\_Acquire\_Release.</b> Acquire an enhanced oplock and then release it.

-   <mark type="bullet_intro">Enhanced\_Acquire\_Release\_Opkey.</b> Acquire an enhanced oplock by using an oplock key.

-   <mark type="bullet_intro">Legacy\_Break\_L2.</b> Break an L2 oplock with various file operations.

-   <mark type="bullet_intro">Enhanced\_Break\_R.</b> Break an R oplock with various file operations.

-   <mark type="bullet_intro">Legacy\_Break\_L1.</b> Break an L1 oplock with various file operations.

-   <mark type="bullet_intro">Enhanced\_Break\_RW.</b> Break an RW oplock with various file operations.

-   <mark type="bullet_intro">Enhanced\_Break\_RH.</b> Break an RH oplock with various file operations.

-   <mark type="bullet_intro">Legacy\_Break\_Batch.</b> Break a Batch oplock with various file operations.

-   <mark type="bullet_intro">Enhanced\_Break\_RWH.</b> Break an RWH oplock with various file operations.

-   <mark type="bullet_intro">EnhancedMultipleSharedOplocks.</b> Acquire multiple enhanced oplocks on the same file.

-   <mark type="bullet_intro">EnhancedUpgradeOplocks.</b> Upgrade an enhanced oplock.

-   <mark type="bullet_intro">EnhancedOplockBreakAckOtherHandle.</b> Acknowledge an enhanced oplock break on another handle that opened via the same key.

-   <mark type="bullet_intro">EnhancedOplockStealthUpgrade.</b> Acknowledge an oplock break and request an upgrade-level oplock.

-   <mark type="bullet_intro">AtomicOplocksVariation.</b> Atomic open (open a handle by using **CreatOptions=FILE\_OPEN\_REQUIRING\_OPLOCK**).

-   <mark type="bullet_intro">AtomicOplocksNonIntrusiveVariation.</b> Atomic open when an oplock already exists on the file.

-   <mark type="bullet_intro">BreakingRHOplocks.</b> Conduct special operations to break an RH oplock.

## Troubleshooting

For generic troubleshooting of HLK test failures, see <xref rid="p_hlk.troubleshooting_windows_hlk_test_failures">Troubleshooting Windows HLK Test Failures</b>.

For troubleshooting information, see <xref rid="p_hlk_test.troubleshooting_file_system_testing">Troubleshooting File System Testing</b>.

All test cases return Pass or Fail. To review test case details, review the test log from Windows Hardware Lab Kit (Windows HLK) Studio.

To run this test, make sure that the computer has NTFS, CNTFS, FAT16, FAT32, ExFAT, and UDF volumes set up. Also, make sure that the drive letter for this volume is entered in the **&lt;filesystem&gt;\_DRIVE\_LETTER** parameters.
