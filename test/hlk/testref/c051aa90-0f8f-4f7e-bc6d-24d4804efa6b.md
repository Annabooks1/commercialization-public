# Hybrid Priority Zero Test

<conditional_block> <conditions> <docset value="standalone"></docset> </conditions>

>[!NOTE]
You can find the latest version of this test documentation on MSDN at the following link:

-   <xref hlink="http://msdn.microsoft.com/en-us/library/windows/hardware/cb611ba4-fa07-4c53-8f2e-c59cca2963bf">http://msdn.microsoft.com/en-us/library/windows/hardware/cb611ba4-fa07-4c53-8f2e-c59cca2963bf</b>


</conditional_block> This test checks to ensure that the drive's behavior at priority zero is correct. This will ensure that the OS can evict data properly from the drive. This is what the test does: o Read data at a fixed length at a random offset on the drive at priority o Write data at a fixed length at a different random offset on the drive at priority o Check the hybrid log to make sure the descriptor values are expected o Write to both regions at priority zero o Check the hybrid log to make sure there is nothing at non-zero priorities.
