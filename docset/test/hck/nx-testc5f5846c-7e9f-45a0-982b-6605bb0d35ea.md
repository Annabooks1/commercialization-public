# NX Test


This test verifies that the system supports No-Execute (NX) at the BIOS and processor level and that Data Execution Prevention (DEP) is not set to “Always Off”.

Drivers must not run code out of the stack, paged pool, and session pool. Also, drivers must not fail to load when Physical Address Extension (PAE) mode is enabled, which is a requirement of NX.

## Running the test

## Troubleshooting


For troubleshooting information, see [Troubleshooting System Fundamentals Testing](troubleshooting-system-fundamentals-testing.md).

This test returns Pass or Fail. To review test details, review the test log from Windows Hardware Certification Kit (Windows HCK) Studio.
