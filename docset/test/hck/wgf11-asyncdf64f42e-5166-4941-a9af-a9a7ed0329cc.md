# WGF11 Async


This automated test verifies D3D graphics driver/hardware conformance for query and other asynchronous features that are not specifically tested elsewhere.

Specifically, this test features the following characteristics:

-   Separate group for each query type

-   Queries tested for asynchronous (non-blocking) execution and correct values

-   Occlusion tested by rendering quads repeatedly with different depth functions

-   Predicates tested with all known predication functions (to ensure that they are predicated)

-   Counters tested with random amounts of geometry being rendered

-   Possibly considers issues with clipping in regards to pipeline statistics

This topic applies to the following test jobs:

-   WGF11 Async

-   WGF11 Async - DeferredContext

-   WGF11 Async - DeferredContext (WoW64)

-   WGF11 Async (WoW64)

## Running the test


Before you run the test, complete the test setup as described in the test requirements: [Graphic Adapter or Chipset Testing Prerequisites](graphic-adapter-or-chipset-testing-prerequisites.md).

## Troubleshooting


For troubleshooting information, see [Troubleshooting Device.Graphics Testing](troubleshooting-devicegraphics-testing.md).

The test might return SKIP if it is run with a feature level that doesn't support the feature being tested. The test might return BLOCKED if there is an uncaught exception (framework catches it at the end and logs it). To review test details, review the test log from the DTM Studio.
